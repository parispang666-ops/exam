# 编辑考试-人员-下级单位 PRD

## 1. 文档范围

本文档仅描述"编辑考试-下级单位"页面的考生范围维护能力，包括考生范围切换、已选人员回显与移除、一键清理离职人员、部门范围维护、部门人员数量动态更新、免考人员配置、人员选择弹窗、以及保存覆盖人员快照。

原有考试基本信息步骤（步骤1）、考试设置步骤（步骤3）、步骤条流转、顶部导航栏、侧边菜单等通用布局能力沿用既有系统规则，本文不重复说明。

**与集团版的核心差异**：下级单位编辑考试仅限本单位及下属部门范围内维护考生范围，不涉及跨公司选择，人员列表不按公司分组展示。

> 本页面PRD需配合[《考试功能-整体PRD》](./考试功能-整体PRD.md)阅读，全局业务规则、数据模型、角色权限定义和核心流程参见整体PRD。

## 2. 使用角色

- **下级单位管理员**：在下级单位端编辑本单位自建考试，进入"考生范围"步骤维护已选人员、部门范围和免考人员，清理离职/禁用人员，并保存覆盖人员快照。
- **系统**：保存后根据当前考试对象覆盖人员快照，作为后续草稿发布前人员变更校验的基准。

## 3. 前置条件

- 用户已登录下级单位端系统
- 用户具有本单位考试管理相关权限
- 当前考试为本单位自建考试，且处于可编辑状态（草稿）
- 上级下发考试不支持编辑考生范围
- 通过考试列表进入编辑考试流程，已完成"基本信息"步骤

## 5. 功能清单

| 模块 | 功能 | 功能描述 |
|------|------|----------|
| 考生范围 | 范围切换 | 支持"仅部分人员"和"组织内全部人员"两种模式切换，编辑时默认继承创建考试时的选择。 |
| 考生范围 | 已选人员回显 | 以标签列表展示已选人员，离职/禁用人员标注"（离职）"，支持查看已选、单个移除、全部移除。 |
| 考生范围 | 一键清理离职人员 | 批量移除所有离职/禁用人员，保留在职/启用人员，并刷新已选人数和标签列表。 |
| 考生范围 | 部门范围回显 | 展示已选部门标签，支持查看已选、单个移除、全部移除。 |
| 考生范围 | 部门人数动态更新 | 编辑考试时基于最新人员主数据重新计算已选部门人数，覆盖创建时的人数快照。 |
| 考生范围 | 免考人员配置 | 支持查看和编辑免考人员与免考部门范围，支持查看已选、单个移除、全部移除。 |
| 人员选择器 | 人员/部门选择 | 通过 iframe 加载下级单位人员选择器，限定本单位及下属部门范围，支持选人员、选部门、查看已选、确认返回。 |
| 考生范围 | 批量导入考生 | 在选择按钮旁提供批量导入入口，支持下载模板、上传 Excel、校验并自动合并到已选考生列表。详见[《批量导入考生 PRD》](./批量导入考生.md)。 |
| 人员快照 | 编辑覆盖快照 | 保存编辑后覆盖原创建考试时人员快照，作为后续发布校验基准。 |

## 6. 需求说明

### 6.1 考试对象范围切换

| 规则项 | 页面要求 |
|--------|----------|
| 模式类型 | 支持"仅部分人员"和"组织内全部人员"。 |
| 默认值 | 编辑时默认继承创建考试时所选模式。 |
| 仅部分人员 | 展示"选择"按钮、指定用户区域、部门范围区域（有数据时）、免考人员区域。 |
| 组织内全部人员 | 隐藏"选择"按钮、指定用户区域、部门范围区域、免考人员区域。 |
| 切换回仅部分人员 | 恢复展示所有区域，已选数据不丢失。 |

### 6.2 指定用户区域

| 展示项 | 规则 |
|--------|------|
| 人员列表 | 已选人员以标签列表直接展示（不按公司分组，因所有人员均在本单位范围内），最多展示 10 个人员标签，超出部分显示"查看更多N项"按钮。 |
| 离职标识 | 离职/禁用人员姓名后标注"（离职）"，红色字体展示。在职/启用人员不展示状态标识。 |
| 单个移除 | 点击人员标签上的 × 按钮，移除该人员，同步刷新人数和标签列表。 |
| 全部移除 | 点击"全部移除"按钮，清空所有指定用户和部门范围数据，并刷新人数。 |
| 查看已选 | 点击"查看已选"按钮，打开人员选择弹窗的已选视图（人员 Tab + 部门 Tab），展示当前已选人员和部门清单。 |
| 选择入口 | 点击"选择"按钮，打开下级单位人员选择弹窗，限定在本单位及下属部门范围内选人和选部门。选择按钮旁展示"共计人员：X人"，基于已选人员和部门解析后去重统计。 |
| 空状态 | 无已选人员时不展示指定用户区域。 |
| 无已选人员/部门时 | "查看已选"和"全部移除"按钮置灰禁用。 |

### 6.3 一键清理离职人员

| 规则项 | 规则 |
|--------|------|
| 按钮位置 | 位于"人员"标题区域，"一键清理离职人员"按钮与人数并排。 |
| 触发条件 | 已选人员中存在离职/禁用人员时按钮可用，否则置灰禁用。 |
| 清理范围 | 仅移除状态为"离职"或"禁用"的已选人员。 |
| 保留范围 | 状态为"启用"的在职人员继续保留在已选列表。 |
| 清理结果 | 移除后立即刷新已选人数和人员标签列表，同步更新"查看已选"和"全部移除"按钮状态。 |
| 清理后保存 | 清理操作仅在前端完成，需点击"保存并下一步"后正式覆盖人员快照。 |

### 6.4 部门范围区域

| 展示项 | 规则 |
|--------|------|
| 可见性 | 仅在已选部门范围有数据时展示。 |
| 部门列表 | 已选部门以标签列表展示（不按公司分组，所有部门均归属于本单位）。 |
| 部门标签 | 展示格式：部门名称。 |
| 单个移除 | 点击部门标签上的 × 按钮，移除该部门，同步刷新部门范围和指定用户区域（部门移除后连带的人员也可能变化）。 |
| 全部移除 | 点击"全部移除"按钮，清空所有部门范围。 |
| 查看已选 | 点击"查看已选"按钮，打开人员选择弹窗的已选视图（部门 Tab），展示当前已选部门清单。 |

### 6.5 部门人员数量动态更新

| 规则项 | 规则 |
|--------|------|
| 触发时机 | 进入编辑考试页面加载已选部门时、点击"选择"按钮打开人员选择弹窗时、进入"查看已选"弹窗时。 |
| 更新场景 | 部门人数需基于当前最新人员主数据重新计算，覆盖创建考试时记录的人数。 |
| 场景1：有人离职 | 部门内原有人员离职 → 部门人数减少，弹窗和已选区域均需展示更新后的人数。 |
| 场景2：有人禁用 | 部门内原有人员被禁用 → 部门人数减少，弹窗和已选区域均需展示更新后的人数。 |
| 场景3：有人移入 | 新人员移入该部门 → 部门人数增加，弹窗和已选区域均需展示更新后的人数。 |
| 场景4：停用改启用 | 部门内原有停用/禁用人员改为启用 → 部门人数增加，弹窗和已选区域均需展示更新后的人数。 |
| 示例 | 新增考试时 A 部门有 5 人 → 编辑时 A 部门变为 6 人（1 人移入） → 弹窗和已选区域人数展示为 6 人。 |
| 影响范围 | 人员选择弹窗中部门列表的人数展示、已选部门标签的人数展示、"查看已选"弹窗中部门对应的人数展示。 |

### 6.6 免考人员区域

| 项目 | 规则 |
|------|------|
| 选择入口 | 点击"免考人员"区域的"+ 选择"按钮，打开下级单位人员选择弹窗，用于选择免考对象。选择按钮旁展示"共计人员：X人"，基于已选免考人员和免考部门解析后去重统计。 |
| 免考对象 | 支持选择免考人员和免考部门范围，均限定在本单位及下属部门范围内。 |
| 已选回显 | 存在免考对象时，顶部展示"免考人员（N人，M个部门）"，下方展示已选人员和已选部门分组标签。 |
| 已选人员 | 以标签列表展示（不按公司分组），离职/禁用人员标注"（离职）"。 |
| 已选部门 | 以标签列表展示部门名称。 |
| 查看已选 | 存在免考对象时，"查看已选"按钮可用，点击打开人员选择弹窗查看免考范围。 |
| 全部移除 | 存在免考对象时，"全部移除"按钮可用，点击清空所有免考人员和免考部门。 |
| 空状态 | 无免考对象时，不展示已选区域，"查看已选"和"全部移除"按钮置灰禁用。 |

### 6.7 人员选择弹窗

| 规则项 | 规则 |
|--------|------|
| 打开方式 | 通过 iframe 加载`人员选择-下级单位.html`页面，以 embed 模式嵌入弹窗。 |
| 弹窗尺寸 | 宽度最大 1100px、高度最大 90vh，自适应视窗。 |
| 弹窗关闭 | 支持点击蒙层、按 ESC 键、iframe 内发送关闭消息三种方式关闭。 |
| 数据通信 | 通过 `postMessage` 与 iframe 内页面通信，初始化时传入已选人员 ID 列表、已选部门 ID 列表、当前激活 Tab 和查看已选参数。 |
| 确认返回 | iframe 内点击确认后，通过 `postMessage` 发送 `person-picker:confirm` 消息，携带选定的人员和部门数据。 |
| 取消/关闭 | 关闭弹窗时不更新页面当前已选结果。 |
| 三种入口 | 1. 指定用户的"选择"按钮；2. 免考人员的"选择"按钮；3. 任意区域的"查看已选"按钮。 |
| 数据去重 | 返回后，系统自动对人员和部门进行去重，确保同一人员只进入一次考生范围。 |
| 范围限制 | 人员选择器限定在本单位及下属部门范围内，不得选择平级单位或上级单位的人员和部门。 |

### 6.8 保存校验与快照覆盖

| 规则项 | 规则 |
|--------|------|
| 保存校验 | "仅部分人员"模式下，点击"保存并下一步"时校验至少选择了一名指定用户或一个部门，否则提示"请至少选择一名指定用户、一个部门或一个岗位"。 |
| "组织内全部人员"模式 | 无需校验已选人员，可直接保存。 |
| 快照覆盖 | 保存成功后，系统基于当前考试对象重新生成人员快照，覆盖创建考试或上一次编辑保存时生成的快照。 |
| 快照内容 | 快照需包含人员 ID、姓名、部门、人员状态（启用/离职/禁用）等信息。与集团版不同，下级单位快照不含"公司"字段。 |
| 后续用途 | 编辑覆盖后的快照作为草稿发布前人员变更校验的最新基准。 |
| 保存后行为 | 进入下一步"考试设置"步骤。 |

### 6.9 批量导入考生

在"仅部分人员"模式下，"选择"按钮右侧展示"批量导入"次级按钮。

| 规则项 | 说明 |
|--------|------|
| 入口位置 | "选择"按钮右侧，"仅部分人员"模式下可见 |
| 模板字段 | 姓名（必填，最大10字符）、手机号（必填，11位数字以1开头） |
| 文件格式 | .xlsx / .xls，原型阶段模板为 `.csv` |
| 单次上限 | 1000 行 |
| 导入结果 | 成功后自动合并到已选考生列表；失败时支持下载错误文件 |
| 去重规则 | Excel 内重复及与已选考生重复均自动跳过 |

详细规则参见[《批量导入考生 PRD》](./批量导入考生.md)。

## 7. 交互设计

### 7.1 范围切换

- 点击"仅部分人员"卡片，展开所有人员选择相关区域。
- 点击"组织内全部人员"卡片，折叠所有人员选择、部门范围、免考人员区域。
- 切换模式时平滑过渡，已选数据在模式间保持不丢失。

### 7.2 已选人员操作

- 点击人员标签上的 × 按钮，仅移除该人员，实时刷新计数和标签列表。
- 点击"全部移除"，弹出确认后清空所有指定用户和部门范围。
- 点击"一键清理离职人员"，即时移除所有离职/禁用人员，无需二次确认。
- "查看已选"和"全部移除"在无已选数据时置灰不可点击。
- 人员标签列表不按公司分组，直接平铺展示。

### 7.3 部门人数更新

- 页面初始加载和每次打开人员选择弹窗时，基于后端最新人员主数据拉取部门人数。
- 已选部门和弹窗中的部门人数展示必须与最新数据一致。

### 7.4 免考人员操作

- 点击"+ 选择"按钮打开弹窗选择免考人员/部门，确认后回写已选结果。
- 已选免考人员和部门标签支持单个移除。
- 免考人员也展示离职标识，但不提供"一键清理离职人员"按钮。
- 未选择免考对象时，"查看已选"和"全部移除"按钮置灰禁用。

### 7.5 保存操作

- 点击"保存并下一步"时，先执行保存校验，通过后保存数据并覆盖快照。
- 点击"上一步"时，返回"基本信息"步骤（步骤1），不触发快照覆盖。

## 8. 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 仅部分人员但未选择考试对象 | 点击保存时弹出提示"请至少选择一名指定用户、一个部门或一个岗位"。 |
| 人员选择弹窗加载失败 | 弹窗内展示加载提示，持续尝试加载 iframe，不阻塞页面其他操作。 |
| 人员选择弹窗取消或关闭 | 不更新当前页面已选结果，保留原有已选数据。 |
| 当前区域无已选结果 | "查看已选"和"全部移除"按钮置灰禁用。 |
| 无离职/禁用人员 | "一键清理离职人员"按钮置灰禁用。 |
| 保存失败 | 提示具体错误信息，保留已编辑内容，用户可再次尝试保存。 |
| 部门人数接口异常 | 部门人数展示上次缓存值或显示"--"，不影响选择操作，后续重试获取。 |
| 下级单位越权选择 | 人员选择器本身限制选择范围，不返回超出本单位及下属部门的人员或部门数据。 |

## 9. 验收标准

| 验收项 | 验收标准 |
|--------|----------|
| 范围切换 | 仅部分人员/组织内全部人员两种模式可正常切换，切换后区域显隐正确，数据不丢失。 |
| 已选人员回显 | 已选人员以标签列表展示，人数正确，离职/禁用人员标注"（离职）"，人员列表不按公司分组。 |
| 一键清理离职人员 | 仅移除离职/禁用人员，保留在职人员，计数和标签同步刷新。 |
| 单个移除人员 | 点击 × 移除单个人，计数减1，标签消失。 |
| 全部移除 | 清空所有指定用户和部门，按钮同步置灰。 |
| 查看已选 | 点击"查看已选"打开弹窗，展示当前已选人员和部门。 |
| 部门范围回显 | 已选部门以标签列表展示，支持查看已选、单个移除、全部移除。 |
| 部门人数动态更新 | 编辑时部门人数基于最新主数据更新；有人离职/禁用/移入/启用变化时，弹窗和已选区域人数正确变化。 |
| 免考人员配置 | 支持选择/回显免考人员和部门，支持查看已选和全部移除，空状态时按钮置灰。 |
| 人员选择弹窗 | 弹窗正常加载、数据通信正常、确认后回写正确、取消不更新数据、范围限定在本单位及下属部门。 |
| 保存校验 | 仅部分人员未选任何对象时提示校验信息，组织内全部人员无校验。 |
| 快照覆盖 | 保存后生成新快照覆盖旧快照，快照内容包含人员ID、姓名、部门、状态（不含公司）。 |
| 上一步 | 点击返回基本信息步骤，不触发保存和快照覆盖。 |
| 权限范围 | 人员选择器不返回超出本单位及下属部门的选人结果。 |

## 10. 与集团版差异对照

| 差异项 | 集团版 | 下级单位版 |
|--------|--------|------------|
| 使用角色 | 集团管理员 | 下级单位管理员 |
| 数据范围 | 集团内跨公司人员、部门 | 本单位及下属部门范围内人员、部门 |
| 人员列表分组 | 按公司分组展示，每组展示公司名称和人数 | 不按公司分组，直接平铺展示标签列表 |
| 人员选择器 | `人员选择-集团.html`，支持跨公司选择 | `人员选择-下级单位.html`，限定本单位及下属部门 |
| 快照内容 | 包含人员ID、姓名、公司、部门、人员状态 | 包含人员ID、姓名、部门、人员状态（不含公司） |
