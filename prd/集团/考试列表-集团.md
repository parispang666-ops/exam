# 考试列表-集团 PRD

## 1. 文档范围

本文档仅描述"考试列表-集团"本次新增功能范围，包括集团自建考试口径、草稿考试发布、发布前人员变更校验和发布弹窗展示规则。

原有列表查询、分页、复制、删除、批量删除、管理、统计等基础能力沿用既有规则，本文不重复说明。

> 本页面PRD需配合[《考试功能-整体PRD》](./考试功能-整体PRD.md)阅读，全局业务规则、数据模型、角色权限定义和核心流程参见整体PRD。

## 2. 使用角色

- 集团管理员：在集团端查看集团自建考试，对草稿考试执行发布，并确认发布前人员变更信息。
- 系统：在草稿发布前比对当前考试对象与创建或编辑考试时的人员快照，并给出人员变更结果。

## 3. 功能清单

| 模块 | 功能 | 功能描述 |
|------|------|----------|
| 考试列表 | 集团自建考试展示 | 列表展示集团端自建考试，作为集团考试管理入口。 |
| 行内操作 | 草稿发布 | 草稿状态考试展示"发布"操作，非草稿考试不展示草稿发布入口。 |
| 发布校验 | 人员快照比对 | 点击发布后，系统比对当前考试对象与创建或编辑考试时的人员快照。 |
| 发布弹窗 | 人员变更确认 | 存在人员变更时展示发布弹窗，管理员确认后继续发布。 |
| 发布弹窗 | 变更分类查看 | 支持按"全部 / 已移除 / 已新增"查看人员变更及人数。 |
| 发布弹窗 | 分页查看 | 人员变更列表支持分页展示，发布说明需固定在当前弹窗可视区域内。 |

## 4. 需求说明

### 4.1 考试列表新增规则

- 页面数据范围为集团自建考试。
- 草稿考试支持发布操作。
- 点击"新增"进入集团新增考试页面。
- 点击"编辑"进入集团编辑考试页面。

### 4.2 草稿发布规则

- 仅草稿状态考试支持发布。
- 点击"发布"后，系统先执行人员快照校验。
- 校验基准为创建考试或最近一次编辑考试后生成的人员快照。
- 如果未检测到人员变更，考试可直接发布。
- 如果检测到人员变更，需先展示发布弹窗，管理员点击弹窗内"发布"后才正式发布。
- 管理员点击取消、关闭弹窗或点击遮罩关闭时，不发布考试，考试保持草稿状态。
- 发布成功后，考试进入已发布后的业务流程；列表状态更新为未开始。

### 4.3 发布前人员变更校验

发布前需覆盖以下校验规则：

| 变更类型 | 考试对象 | 触发条件 | 展示要求 |
|----------|----------|----------|----------|
| 移除 | 按人员 | 已选人员存在离职或停用人员 | 展示系统即将移除的人员列表和数量。 |
| 移除 | 按部门范围 | 部门内存在离职、停用或转出部门人员 | 展示系统即将移除的人员列表和数量。 |
| 新增 | 按部门范围 | 部门内存在转入部门人员或启用人员 | 展示新进入考试对象的人员列表和数量。 |

### 4.4 发布弹窗展示规则

#### 4.4.1 弹窗基础信息

| 项目 | 展示规则 |
|------|----------|
| 弹窗标题 | 展示为"发布"。 |
| 检测提示 | 固定展示：`【系统检测】与您创建/编辑考试时相比，人员列表已产生变更，变更信息如下：` |
| 操作按钮 | 底部展示"取消"和"发布"。点击"取消"关闭弹窗并保留草稿状态；点击"发布"确认发布考试。 |

#### 4.4.2 变更分类切换

| 分类 | 展示内容 | 人数统计 |
|------|----------|----------|
| 全部 | 展示移除和新增的全部人员变更。 | 展示全部变更人员数量。 |
| 已移除 | 展示系统即将移除的人员。 | 展示移除人员数量。 |
| 已新增 | 展示新进入考试对象的人员。 | 展示新增人员数量。 |

切换分类后，列表内容和分页状态同步刷新，页码回到第一页。

#### 4.4.3 人员变更列表

| 展示项 | 展示规则 |
|--------|----------|
| 序号 | 按当前分类下的结果顺序连续展示。 |
| 姓名 | 展示人员姓名。离职、停用人员姓名后标注"（离职）"。 |
| 所属公司 | 展示人员所属公司名称。 |
| 部门 | 展示人员所属部门。 |
| 操作类型 | 展示"移除"或"新增"。 |

转入部门、转出部门、启用人员不展示姓名状态标识。

#### 4.4.4 分页与空状态

| 场景 | 展示规则 |
|------|----------|
| 分页展示 | 人员变更列表支持分页，默认每页展示 5 条。 |
| 分页切换 | 分页切换后，序号按当前分类下的全部结果顺序连续展示。 |
| 无数据 | 当前分类下无人员变更时，列表展示"暂无人员变更"。 |

### 4.5 发布说明

发布弹窗下方需固定展示以下说明，翻页时不被列表内容挤出当前弹窗可视区域：

**考试发布后，新入职的人员不自动纳入考试中。考试发布后，离职人员不影响考试流程，但会影响应考人数。**

## 5. 交互设计

### 5.1 发布按钮

- 草稿考试行内展示"发布"入口。
- 点击发布后先执行发布前人员变更校验。

### 5.2 发布弹窗

- 点击发布且存在人员变更时打开弹窗。
- 点击弹窗右上角关闭、底部取消或遮罩区域，关闭弹窗且不改变考试状态。
- 点击弹窗底部"发布"后，确认发布考试，并刷新列表状态。
- 弹窗内切换"全部 / 已移除 / 已新增"时，只改变弹窗内列表展示，不直接改变考试状态。
- 弹窗分页只影响当前分类下的人员变更列表，不影响发布说明和底部操作按钮展示。

## 6. 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 发布弹窗关闭 | 不发布考试，考试继续保持草稿状态。 |
| 当前分类无人员变更 | 变更列表展示"暂无人员变更"。 |

## 7. 验收标准

| 验收项 | 验收标准 |
|--------|----------|
| 集团列表范围 | 页面展示集团自建考试。 |
| 草稿发布入口 | 草稿考试展示"发布"操作，点击后先执行人员快照校验。 |
| 无变更发布 | 未检测到人员变更时，可直接发布考试。 |
| 有变更弹窗 | 检测到人员变更时，必须展示发布弹窗。 |
| 分类切换 | 发布弹窗支持"全部 / 已移除 / 已新增"切换，并展示各分类人数。 |
| 校验规则 | 发布前校验必须覆盖按人员移除、按部门范围移除、按部门范围新增三类场景。 |
| 人员标识 | 仅离职、停用人员姓名后展示"（离职）"。 |
| 所属公司字段 | 发布弹窗人员变更列表展示"所属公司"字段。 |
| 分页展示 | 发布弹窗人员变更列表支持分页，默认每页 5 条。 |
| 固定说明 | 发布说明在弹窗当前可视区域内固定可见。 |
| 取消发布 | 取消或关闭弹窗后，考试保持草稿状态。 |
| 确认发布 | 点击弹窗"发布"后，考试发布成功，列表状态更新。 |
