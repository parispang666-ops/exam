# 考试统计-集团 PRD

## 1. 文档范围

本文档仅描述"考试统计-集团"页面中与**统计范围、人员状态、所属公司字段、公司汇总视图、部门统计人员过滤**相关的需求。普通统计卡片（及格率/参与率/分数概览）、考试记录列表、导出、分页等基础统计能力沿用既有考试统计规则，本文不重复说明。

> 与下级单位统计版本的主要差异：
> 1. 考生分析/考试记录列表新增"所属公司"列
> 2. 部门统计支持"部门明细 / 公司汇总"视图切换
> 3. 集团租户下人员来自多家公司，部门按公司级联筛选

页面原型文件：`考试统计-集团.html`

> 本页面PRD需配合[《考试功能-整体PRD》](./考试功能-整体PRD.md)阅读，全局业务规则、数据模型、角色权限定义和核心流程参见整体PRD。

## 2. 使用角色

- **集团管理员**：在集团范围内跨公司查看考生分析、考试记录、部门统计（含公司汇总），支持按人员状态筛选，部门统计默认仅统计在职人员。

## 3. 前置条件

- 用户已登录集团管理端
- 当前考试已发布，已生成完整的人员快照和考试记录
- 从考试列表点击"管理"入口进入统计页面
- 考试对象可能来自多家公司

## 5. 功能清单

| 模块 | 功能 | 功能描述 |
|------|------|----------|
| 统计范围 | 全量人员保留 | 考试发布时的全部人员均保留在统计列表中，离职、停用后人员记录不隐藏、不移除。 |
| 考生分析 | 所属公司列 + 人员状态 | 列表新增"所属公司"列和"人员状态"列；查询条件新增公司+部门级联筛选、"人员状态"筛选。 |
| 考试记录 | 所属公司列 + 人员状态 | 列表新增"所属公司"列和"人员状态"列；查询条件新增公司+部门级联筛选、"人员状态"筛选。 |
| 部门统计 | 视图切换 + 只统计在职人员 | 支持"部门明细/公司汇总"视图切换；筛选区新增"只统计在职人员"勾选项，默认勾选。部门明细按公司→部门多级树展示，公司汇总按公司扁平汇总。 |

## 6. 需求说明

### 6.1 统计范围：全量人员保留

| 规则项 | 页面要求 |
|--------|----------|
| 人员基准 | 统计列表基于考试发布时生成的人员快照，包含考试发布时的全部应考人员。 |
| 离职人员 | 考试发布后离职的人员，其姓名仍然保留在考生分析和考试记录列表中，通过"人员状态"列标注为"离职"。 |
| 停用人员 | 考试发布后被停用的人员，其姓名仍然保留在考生分析和考试记录列表中，通过"人员状态"列标注为"停用"。 |
| 不隐藏规则 | 离职、停用人员记录不被系统自动移除或隐藏，用户可通过查询条件和列表直接查看。 |
| 统计口径 | 离职、停用人员在考生分析、考试记录中正常展示；在部门统计中受"只统计在职人员"勾选项控制。 |

### 6.2 考生分析 / 考试记录：新增所属公司列与人员状态

#### 6.2.1 查询条件新增

与下级单位版本相比，集团版查询条件有以下差异：

| 规则项 | 下级单位 | 集团版 |
|--------|----------|--------|
| 部门筛选 | 部门下拉（本单位及下属部门级联） | **公司+部门级联**：先选择公司，再选择该公司下的部门 |
| 人员状态 | 下拉（请选择/在职/离职/停用） | 同下级单位 |
| 其他筛选 | 姓名、完成状态、是否及格等 | 同下级单位 |

公司+部门级联规则：

| 规则项 | 规则 |
|--------|------|
| 组件类型 | 两级级联选择器：左侧公司列表，右侧部门列表 |
| 默认值 | 请选择（不限制公司和部门） |
| 交互 | 选择公司后，右侧自动刷新该公司下的部门列表；可直接选择公司或进一步选择部门 |
| 查询逻辑 | 选择公司后查询该公司下全部人员；选择具体部门后查询该部门人员 |

人员状态筛选（与下级单位一致）：

| 规则项 | 规则 |
|--------|------|
| 标签 | 统一使用"人员状态"。 |
| 下拉选项 | 请选择（默认）、在职、离职、停用。 |
| 默认值 | 请选择，即不按人员状态筛选，展示全部人员。 |
| 查询逻辑 | 选择具体状态值后点击"查询"，过滤出对应人员状态的记录。 |

#### 6.2.2 列表新增列

| 规则项 | 下级单位 | 集团版 |
|--------|----------|--------|
| 列顺序 | 姓名 → 部门 → 人员状态 | 姓名 → **所属公司** → 部门 → 人员状态 |

所属公司列规则：

| 规则项 | 规则 |
|--------|------|
| 列名 | 所属公司 |
| 位置 | 位于"姓名"列之后、"部门"列之前。 |
| 展示内容 | 人员所属公司的名称。 |
| 排序 | 默认按公司分组排列，同公司人员连续展示。 |

人员状态列规则（与下级单位一致）：

| 规则项 | 规则 |
|--------|------|
| 列名 | 人员状态 |
| 位置 | 位于"部门"列之后。 |
| 展示风格 | 使用彩色标签展示：在职—绿色标签、离职—红色标签、停用—橘黄色标签。 |
| 数据来源 | 取自人员快照中记录的人员状态。缺失显示"--"。 |

#### 6.2.3 人员状态枚举与样式

| 状态值 | 标签样式 | 说明 |
|--------|----------|------|
| 在职 | 绿色背景 + 绿色文字（`person-state`） | 人员当前在职/启用。 |
| 离职 | 红色背景 + 红色文字（`person-state left`） | 人员已离职，姓名保留在列表中。 |
| 停用 | 橘黄色背景 + 橘黄色文字（`person-state disabled`） | 人员已停用/禁用，姓名保留在列表中。 |

### 6.3 部门统计：视图切换 + 只统计在职人员

#### 6.3.1 视图切换

集团版部门统计包含两个视图：

| 视图名称 | 说明 | 展示结构 |
|----------|------|----------|
| **部门明细**（默认） | 按公司→部门多级树结构展示 | 公司行（加粗）+ 部门行（缩进），支持层级展开/收起 |
| **公司汇总** | 按公司扁平汇总展示 | 每个公司一行，各指标为公司下所有部门数据的汇总 |

视图切换规则：

| 规则项 | 规则 |
|--------|------|
| 组件类型 | 按钮组切换（`部门明细` / `公司汇总`）。 |
| 默认视图 | **部门明细**。 |
| 切换行为 | 点击按钮即时切换视图，保留当前筛选条件。 |
| 筛选条件 | 两种视图共用同一个部门名称搜索框和"只统计在职人员"勾选项。 |

#### 6.3.2 部门明细视图

| 规则项 | 规则 |
|--------|------|
| 展示结构 | 公司→部门→子部门的多级树结构，公司行为父级，其下部门缩进展示。 |
| 公司行样式 | 加粗字体、背景色区分（`background: #F8FBFF`），行首可选择展开/收起。 |
| 默认展开 | 部门明细默认展开第一级公司下的部门，其余公司可手动展开。 |
| 列名 | "部门名称"列在部门明细视图中同时展示公司和部门名称。 |
| 指标计算 | 公司行指标 = 该公司下所有部门指标的汇总（应考、实考等直接求和；率值按汇总后重新计算）。 |

#### 6.3.3 公司汇总视图

| 规则项 | 规则 |
|--------|------|
| 展示结构 | 扁平表格，每个公司单独一行。 |
| 列名 | 表头列名变更为"公司名称"。 |
| 指标计算 | 与部门明细中公司行的计算口径一致，为该公司下各部门的汇总。 |
| 排序 | 默认按公司名称排序，支持列表点击排序（待确认）。 |

#### 6.3.4 只统计在职人员

| 规则项 | 规则 |
|--------|------|
| 组件类型 | 复选框 + 文字标签。 |
| 标签文案 | "只统计在职人员"。 |
| 默认状态 | **勾选**（`checked`），即默认只统计在职/启用人员。 |
| 勾选时 | 各指标仅统计人员状态为"在职"的记录，两种视图'均生效。 |
| 取消勾选时 | 统计范围扩大到全部人员，包含在职、离职、停用，两种视图均生效。 |
| 切换行为 | 勾选/取消勾选后，需点击"查询"按钮触发数据刷新，而非即时生效。 |
| 影响范围 | 仅影响部门统计页签，不影响考生分析和考试记录页签的数据展示。 |

## 7. 交互设计

### 7.1 人员状态筛选

- 在考生分析或考试记录页签，选择"人员状态"下拉值并点击"查询"后，列表仅展示对应状态的记录。
- 选择"请选择"并查询，恢复展示全部人员。
- 点击"重置"，人员状态下拉恢复为"请选择"。

### 7.2 公司+部门级联筛选

- 公司列表面板展示本次考试涉及的全部公司。
- 选择公司后，右侧部门面板展示该公司下的部门列表。
- 选择具体部门后，列表按部门过滤；只选公司不选部门，则查询该公司下全部人员。
- 级联面板点击外部区域或重新选择时关闭。

### 7.3 部门明细展开/收起

- 公司行首显示展开/收起图标（箭头），默认展开。
- 点击公司行收起，其下所有部门行隐藏，公司行保留并展示汇总数据。
- 展开时恢复部门行展示。

### 7.4 视图切换

- 部门明细与公司汇总之间切换为即时切换，无需重新查询。
- 切换视图时，筛选条件和"只统计在职人员"勾选状态保留。
- 部门明细视图下表头为"部门名称"，公司汇总视图下表头变更为"公司名称"。

### 7.5 只统计在职人员切换

- 进入部门统计页签时，勾选项默认勾选，视图按在职口径展示。
- 用户取消勾选后点击"查询"，视图刷新为全量口径。
- 重新勾选后点击"查询"，视图恢复为在职口径。
- 切换行为在两个视图（部门明细/公司汇总）下均保持一致。

### 7.6 页签切换

- 在考生分析、考试记录、部门统计三个页签之间切换时，各页签的查询条件和数据状态保持独立。
- 考生分析和考试记录的"人员状态"筛选互不影响。
- 部门统计的"只统计在职人员"勾选状态和视图选择仅在部门统计页签内有效。

## 8. 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 人员快照数据缺失人员状态 | 人员状态列显示"--"，不作为筛选条件参与过滤。 |
| 部门统计全量口径下某公司/部门无数据 | 该公司/部门行正常展示，指标值显示"0"或"0%"。 |
| 公司汇总无匹配数据 | 列表展示空状态提示"暂无数据"。 |
| 部门明细某公司下无部门 | 公司行仍展示，指标值显示"0"或"0%"，不显示子行。 |
| 级联选择器中公司无部门 | 右侧部门面板展示空状态。 |

## 9. 验收标准

| 验收项 | 验收标准 |
|--------|----------|
| 离职人员保留 | 考试发布后离职的人员仍在考生分析和考试记录列表中展示，人员状态显示"离职"。 |
| 停用人员保留 | 考试发布后停用的人员仍在考生分析和考试记录列表中展示，人员状态显示"停用"。 |

| 公司+部门级联 | 考生分析和考试记录的部门筛选为公司+部门两级级联，选择公司后部门列表正确过滤。 |
| 考生分析人员状态列 | 列表包含"人员状态"列，展示在职/离职/停用，样式与枚举一致。 |

| 考试记录人员状态列 | 列表包含"人员状态"列，展示在职/离职/停用，样式与枚举一致。 |
| 考试记录人员状态筛选 | 查询条件包含"人员状态"下拉，选择后查询正确过滤。 |
| 部门统计默认视图 | 进入部门统计页签时，默认展示"部门明细"视图。 |
| 部门明细展示 | 公司行为加粗父级行，部门行缩进展示，支持展开/收起。 |
| 公司汇总展示 | 点击"公司汇总"按钮切换为扁平公司列表，各指标为公司下部门汇总。 |
| 视图切换保留条件 | 切换视图时保留当前筛选条件、"只统计在职人员"勾选状态，表头正确变更。 |
| 部门统计默认勾选 | 进入部门统计页签时，"只统计在职人员"默认勾选。 |
| 部门统计勾选口径 | 勾选时仅统计在职人员，两种视图均生效。 |
| 部门统计取消勾选 | 取消勾选后点击查询，统计包含全部人员，两种视图均生效。 |
| 页签独立性 | 各页签的筛选条件和数据状态互不影响。 |
