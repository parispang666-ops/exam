# 考试统计-下级单位 PRD

## 1. 文档范围

本文档仅描述"考试统计-下级单位"页面中与**统计范围、人员状态、部门统计人员过滤**相关的需求。普通统计卡片（及格率/参与率/分数概览）、考试记录列表、导出、分页等基础统计能力沿用既有考试统计规则，本文不重复说明。

页面原型文件：`考试统计-下级单位:独立租户.html`

> 本页面PRD需配合[《考试功能-整体PRD》](./考试功能-整体PRD.md)阅读，全局业务规则、数据模型、角色权限定义和核心流程参见整体PRD。

## 2. 使用角色

- **下级单位管理员**：在本单位及下属部门范围内查看考生分析、考试记录、部门统计，支持按人员状态筛选，部门统计默认仅统计在职人员。
- **普通租户管理员**：在当前租户范围内使用相同的统计能力。

## 3. 前置条件

- 用户已登录下级单位或普通租户管理端
- 当前考试已发布，已生成完整的人员快照和考试记录
- 从考试列表点击"管理"入口进入统计页面

## 5. 功能清单

| 模块 | 功能 | 功能描述 |
|------|------|----------|
| 统计范围 | 全量人员保留 | 考试发布时的全部人员均保留在统计列表中，离职、停用后人员记录不隐藏、不移除。 |
| 考生分析 | 人员状态列新增 | 列表新增"人员状态"列，展示在职/离职/停用，查询条件新增"人员状态"筛选。 |
| 考试记录 | 人员状态列新增 | 列表新增"人员状态"列，展示在职/离职/停用，查询条件新增"人员状态"筛选。 |
| 部门统计 | 只统计在职人员 | 筛选区新增"只统计在职人员"勾选项，默认勾选；取消勾选后纳入离职、停用人员。 |

## 6. 需求说明

### 6.1 统计范围：全量人员保留

| 规则项 | 页面要求 |
|--------|----------|
| 人员基准 | 统计列表基于考试发布时生成的人员快照，包含考试发布时的全部应考人员。 |
| 离职人员 | 考试发布后离职的人员，其姓名仍然保留在考生分析和考试记录列表中，通过"人员状态"列标注为"离职"。 |
| 停用人员 | 考试发布后被停用的人员，其姓名仍然保留在考生分析和考试记录列表中，通过"人员状态"列标注为"停用"。 |
| 不隐藏规则 | 离职、停用人员记录不被系统自动移除或隐藏，用户可通过查询条件和列表直接查看。 |
| 统计口径 | 离职、停用人员在考生分析、考试记录中正常展示；在部门统计中受"只统计在职人员"勾选项控制。 |

### 6.2 考生分析 / 考试记录：新增人员状态

#### 6.2.1 查询条件新增

| 规则项 | 规则 |
|--------|------|
| 位置 | 查询条件区域，位于"部门"筛选之后。 |
| 标签 | 统一使用"人员状态"。 |
| 下拉选项 | 请选择（默认）、在职、离职、停用。 |
| 默认值 | 请选择，即不按人员状态筛选，展示全部人员。 |
| 查询逻辑 | 选择具体状态值后点击"查询"，过滤出对应人员状态的记录。 |

#### 6.2.2 列表新增列

| 规则项 | 规则 |
|--------|------|
| 列名 | 人员状态 |
| 位置 | 位于"部门"列之后、"完成状态/当次作答次数"列之前。 |
| 展示风格 | 使用彩色标签展示：在职—绿色标签、离职—红色标签、停用—橘黄色标签。 |
| 数据来源 | 取自人员快照中记录的人员状态或实时人员主数据状态（待确认具体取数逻辑）。 |

#### 6.2.3 人员状态枚举与样式

| 状态值 | 标签样式 | 说明 |
|--------|----------|------|
| 在职 | 绿色背景 + 绿色文字（`person-state`） | 人员当前在职/启用。 |
| 离职 | 红色背景 + 红色文字（`person-state left`） | 人员已离职，姓名保留在列表中。 |
| 停用 | 橘黄色背景 + 橘黄色文字（`person-state disabled`） | 人员已停用/禁用，姓名保留在列表中。 |

### 6.3 部门统计：只统计在职人员

| 规则项 | 规则 |
|--------|------|
| 位置 | 部门统计页签的查询条件区域，位于"部门"筛选之后、"查询/重置"按钮之前。 |
| 组件类型 | 复选框 + 文字标签。 |
| 标签文案 | "只统计在职人员"。 |
| 默认状态 | **勾选**（`checked`），即默认只统计在职/启用人员。 |
| 勾选时 | 部门统计的各指标（应考人数、实考人数、缺考人数、及格人数、参与率、及格率、最高分、最低分、平均分）仅统计人员状态为"在职"的记录。 |
| 取消勾选时 | 统计范围扩大到全部人员，包含在职、离职、停用人员，各指标基于全量数据重新计算。 |
| 切换行为 | 勾选/取消勾选后，需点击"查询"按钮触发数据刷新，而非即时生效。 |
| 影响范围 | 仅影响部门统计页签，不影响考生分析和考试记录页签的数据展示。 |

## 7. 交互设计

### 7.1 人员状态筛选

- 在考生分析或考试记录页签，选择"人员状态"下拉值并点击"查询"后，列表仅展示对应状态的记录。
- 选择"请选择"并查询，恢复展示全部人员。
- 点击"重置"，人员状态下拉恢复为"请选择"。

### 7.2 只统计在职人员切换

- 进入部门统计页签时，勾选项默认勾选，列表按在职口径展示。
- 用户取消勾选后点击"查询"，列表刷新为全量口径。
- 重新勾选后点击"查询"，列表恢复为在职口径。

### 7.3 页签切换

- 在考生分析、考试记录、部门统计三个页签之间切换时，各页签的查询条件和数据状态保持独立。
- 考生分析和考试记录的"人员状态"筛选互不影响。
- 部门统计的"只统计在职人员"勾选状态仅在部门统计页签内有效。

## 8. 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 人员快照数据缺失人员状态 | 人员状态列显示"--"，不作为筛选条件参与过滤。 |
| 部门统计全量口径下某部门无数据 | 该部门行正常展示，指标值显示"0"或"0%"。 |
| 切换页签时查询结果为空 | 列表展示空状态提示"暂无数据"。 |

## 9. 验收标准

| 验收项 | 验收标准 |
|--------|----------|
| 离职人员保留 | 考试发布后离职的人员仍在考生分析和考试记录列表中展示，人员状态显示"离职"。 |
| 停用人员保留 | 考试发布后停用的人员仍在考生分析和考试记录列表中展示，人员状态显示"停用"。 |
| 考生分析人员状态列 | 列表包含"人员状态"列，展示在职/离职/停用，样式与枚举一致。 |
| 考生分析人员状态筛选 | 查询条件包含"人员状态"下拉，选择后查询正确过滤。 |
| 考试记录人员状态列 | 列表包含"人员状态"列，展示在职/离职/停用，样式与枚举一致。 |
| 考试记录人员状态筛选 | 查询条件包含"人员状态"下拉，选择后查询正确过滤。 |
| 部门统计默认勾选 | 进入部门统计页签时，"只统计在职人员"默认勾选。 |
| 部门统计勾选口径 | 勾选时仅统计在职人员，各指标基于在职人员计算。 |
| 部门统计取消勾选 | 取消勾选后点击查询，统计包含全部人员（在职、离职、停用），各指标基于全量数据重新计算。 |
| 页签独立性 | 各页签的筛选条件和数据状态互不影响。 |
